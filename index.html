Summary :

This study was carried out during my first year of the Master of Economics program at the Université Cote d’Azur as a requirement for the Big Data and Machine Learning course. In this study, we want to forecast the median value of owner-occupied homes in several Boston areas using a collection of 18 characteristics, including crime rate, percentage of residents with lower socioeconomic level, and proximity to highways. We investigate and clean the dataset first, and then we train and test our models using a variety of regression models, including linear regression, decision trees, and random forests. To choose the best model for predicting home prices, we analyze the models’ performances and evaluate the models using metrics like adjusted R-squared and root mean squared error.
The random forest model outperformed the other models, according to the results, having the lowest RMSE and greatest adjusted R-squared. The experiment demonstrates the applicability of machine learning in the study of economics and underlines its potential in predicting home prices.

Introduction

The “BostonHousing2” dataset is one of the many datasets available in R for data exploration and analysis. This dataset contains information on housing prices in the city of Boston, as well as various features of the houses and their environment. The data was collected in the 1970s for a study on air quality in cities. The “BostonHousing2” dataset is often used as an example for applying machine learning techniques and statistical modeling to predict housing prices. The dataset consists of 506 observations and 18 variables, including the median value of owner-occupied homes (in thousands of dollars), the proportion of residential land zoned for lots over 25,000 square feet, nitric oxide concentration, as well as other socio-economic characteristics. The dataset is commonly used in machine learning and data analysis projects to demonstrate the effectiveness of algorithms and models for predicting housing prices.

Data :

The original data are 506 observations on 14 variables, cmedv being the target variable:

crim per capita crime rate by town

zn proportion of residential land zoned for lots over 25,000 sq.ft

indus proportion of non-retail business acres per town

chas Charles River dummy variable (= 1 if tract bounds river; 0 otherwise)

nox nitric oxides concentration (parts per 10 million)

rm average number of rooms per dwelling age proportion of owner-occupied units built prior to 1940

dis weighted distances to five Boston employment centres

rad index of accessibility to radial highways

tax full-value property-tax rate per USD 10,000 ptratio pupil-teacher ratio by town

b 1000(B - 0.63)^2 where B is the proportion of blacks by town

lstat percentage of lower status of the population

medv median value of owner-occupied homes in USD 1000’s

The corrected data set has the following additional columns:

cmedv corrected median value of owner-occupied homes in USD 1000’s

town name of town

tract census tract

lon longitude of census

tract lat latitude of census tract

Regression models

Linear Regression

Let’s attempt a generalized linear regression model first, with the dependent variable cmedv and all the other variables acting as independent variables. Using the training dataset, we train the model. The coefficients of each feature and the intercept for this linear model are listed below. The outcome (cmedv) for the testing dataset is then predicted using the trained model. Calculating the root-mean squared error is a useful metric for evaluating the model’s precision.

fit.lm <- lm(cmedv~.,data = train)

#check coeffs
data.frame(coef = round(fit.lm$coefficients,2))
#predict on test set
pred.lm <- predict(fit.lm, newdata = test)

# Root-mean squared error
rmse.lm <- sqrt(sum((pred.lm - test$cmedv)^2)/
                  length(test$cmedv))

c(RMSE = rmse.lm, R2 = summary(fit.lm)$r.squared)
##      RMSE        R2 
## 4.8131652 0.7372172

We see that the RMSE is 4.8 and the R2 value is 0.74 for this model.

The output variable, cmedv, was skewed to the right, as we have seen. The distribution of cmedv would be normalized by using a log transformation. Let’s run glm with all remaining features as input and log(cmedv) as the output. We can observe that this model’s RMSE value has decreased.

Other regressions

Rpart, Tree-based and Random Forest models

The linear model formula of PRICE (which states that PRICE is the outcome with all other features as input including cmedv, which is our aim) might be used for rpart, tree, and random forest implementations. Looking at the data, we can see that the random forest model has provided the most accurate results to date.

Also, we created a new column to our dataset named “price” that has the same values as the “cmedv” variable. This was done in order to make our study more understandable and since we found that our models performed worse when we used “cmedv” as the target variable explicitly. Our models are better able to capture the links between the other factors in the dataset and the target variable by utilizing “price” as the target variable.

BostonHousing2$PRICE <- BostonHousing2$cmedv
#Creations of classifiers
model1 <- rpart(PRICE ~ ., train)
modeltree <- tree(PRICE ~ .,train)
modelRtree <- randomForest(PRICE ~ . ,train)


#Application of classifiers to the test set
test_model1 <- predict(model1, test)
test_modeltree <- predict(modeltree,test)
test_modelRtree <- predict(modelRtree, test)

#Addition of columns with predictions from the first three classifiers to the test data:
test$model1 <- test_model1 
test$modeltree <- test_modeltree
test$modelRtree <- test_modelRtree
#4) Evaluation of classifiers

#LOSS FUNCTION

# Calculation of MSE, RMSE and MAE models 1, 2 et 3
mse1 <- mean((test$PRICE - test$model1)^2)
rmse1 <- sqrt(mean((test$PRICE - test$model1)^2))
mae1 <- mean(abs(test$PRICE - test$model1))

mse2 <- mean((test$PRICE - test$modeltree)^2)
rmse2 <- sqrt(mean((test$PRICE - test$modeltree)^2))
mae2 <- mean(abs(test$PRICE - test$modeltree))

mse3 <- mean((test$PRICE - test$modelRtree)^2)
rmse3 <- sqrt(mean((test$PRICE - test$modelRtree)^2))
mae3 <- mean(abs(test$PRICE - test$modelRtree))
# Find the model with MSE, RMSE et MAE minimum
best_mse <- min(mse1, mse2, mse3)
best_rmse <- min(rmse1, rmse2, rmse3)
best_mae <- min(mae1, mae2, mae3)

# Conclusion
cat("The model with the smallest MSE is :", ifelse(best_mse == mse1, "model1", ifelse(best_mse == mse2, "modeltree", "modelRtree")), "\n")
## The model with the smallest MSE is : modelRtree
cat("The model with the smallest RMSE is :", ifelse(best_rmse == rmse1, "model1", ifelse(best_rmse == rmse2, "modeltree", "modelRtree")), "\n")
## The model with the smallest RMSE is : modelRtree
cat("The model with the smallest MAE is :", ifelse(best_mae == mae1, "model1", ifelse(best_mae == mae2, "modeltree", "modelRtree")), "\n")
## The model with the smallest MAE is : modelRtree
#Other metrics to evaluate models

# Calculation of R squared
rsq1 <- 1 - sum((test$PRICE - test$model1)^2) / sum((test$PRICE - mean(test$PRICE))^2)
rsq2 <- 1 - sum((test$PRICE - test$modeltree)^2) / sum((test$PRICE - mean(test$PRICE))^2)
rsq3 <- 1 - sum((test$PRICE - test$modelRtree)^2) / sum((test$PRICE - mean(test$PRICE))^2)

# Calculation of Adjusted R squared
adj_rsq1 <- 1 - (1 - rsq1) * (nrow(test) - 1) / (nrow(test) - ncol(test) - 1)
adj_rsq2 <- 1 - (1 - rsq2) * (nrow(test) - 1) / (nrow(test) - ncol(test) - 1)
adj_rsq3 <- 1 - (1 - rsq3) * (nrow(test) - 1) / (nrow(test) - ncol(test) - 1)
if (adj_rsq1 > adj_rsq2 && adj_rsq1 > adj_rsq3) {
  cat("The model with the highest adjusted R-squared is Rpart.\n")
} else if (adj_rsq2 > adj_rsq1 && adj_rsq2 > adj_rsq3) {
  cat("The model with the highest adjusted R-squared is Tree-based 2.\n")
} else if (adj_rsq3 > adj_rsq1 && adj_rsq3 > adj_rsq2) {
  cat("The model with the highest adjusted R-squared is Random Forest.\n")
  cat("The adjusted R-squared is equal to: ", max(adj_rsq1, adj_rsq2, adj_rsq3), "\n")
} else {
  cat("There is a tie for the model with the highest adjusted R-squared.\n")
}

Conclusion

To estimate the home prices in the Boston suburbs, we tested with a variety of linear regression, tree-based, and random forest models. The best model to predict the outcome with the least RMSE was a random forest model with a straightforward linear relationship between the outcome and all input characteristics.
